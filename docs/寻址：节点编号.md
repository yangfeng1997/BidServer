# 寻址：NodeID

本文档记录当前框架的节点寻址规则。NodeID 是服务发现、RouterAgent 路由、RPC 目标选择共同使用的节点身份。

## 编码布局

```text
NodeID uint32 = | 16 bit worldID | 8 bit serverType | 8 bit serverIndex |
调试展示格式：worldID.serverType.serverIndex，例如 111.1.3
```

当前实现位于 `internal/core/nodeid`：

```go
id := nodeid.Encode(worldID, serverType, serverIndex)
worldID, serverType, serverIndex := nodeid.Decode(id.Uint32())
text := nodeid.String(id.Uint32())
id, err := nodeid.Parse("111.1.3")
```

约定：

- NodeID 的传输和存储使用 `uint32` 原值。
- 点分格式只用于日志、调试和文本解析，不作为 wire 格式。
- `worldID` 已编码在 NodeID 内，框架路由不需要额外的 world 维度。
- `serverType` 来自 `protocol/common/node.proto` 的 `ServerType` 枚举。
- `serverIndex` 由部署或配置分配，同一 world、同一 serverType 下必须唯一。

## 进程身份

每个进程启动后应拥有稳定 NodeID：

```text
NodeID = Encode(worldID, serverType, serverIndex)
```

其中：

- `worldID` 表示逻辑世界或分区。
- `serverType` 表示服务类型，例如 `ST_GATESVR`、`ST_LOBBYSVR`、`ST_ROUTERAGENT`。
- `serverIndex` 表示同类服务实例编号。

启动后 NodeID 不应变化。变化意味着旧节点应下线，新节点重新注册。

## 路由语义

框架路由只认 NodeID 和 serverType：

- `At(nodeID)`：使用完整 NodeID 直达指定节点，可跨 world。
- `ByHash(key)`：按目标 serverType 的存活节点集合选择一个节点。
- 默认裸调：按目标 serverType 选择一个可用节点。
- `Broadcast()`：向目标 serverType 的全部存活节点发送。

`At` 是精确寻址；`ByHash`、默认裸调和广播是按 serverType 的成员集合寻址。

## 与 RouterAgent 的关系

RouterAgent 收到业务进程 UDS Handshake 时读取业务进程提交的 NodeID，并通过 `nodeid.Decode` 得到 serverType。成员表以 NodeID 为主键，同时维护 serverType 到节点列表的索引。

etcd 注册信息和 RouterAgent 成员表都应以 NodeID 为节点身份。业务进程之间不直接暴露自己的监听地址，跨进程通信由本机 RouterAgent 代转。
