# P5-A：§9.3 死代码 / 占位清理 设计 Spec

> 路线图依据：[`2026-06-02-implementation-roadmap-P2-P5.md`](2026-06-02-implementation-roadmap-P2-P5.md) §P5「淘汰 / 清理（设计稿 §9.3）」。
> 本切片是 P5 收口的第一片：删除被新架构淘汰的占位与示例协议，使代码库与目标拓扑一致。属机械、低风险、纯沙箱可验。

## 1. 背景与目标

新架构目标拓扑为 gateway / lobby / router / online / match / room（见 `architecture.md`）。仓库里仍残留旧示例体系的占位与协议：`dbsvr`/`loginsvr` 空占位目录、`worldsvr`/`scenesvr` 示例协议（`world.proto` + `gen/world`）、一份零引用的孤儿 `cluster.pb.go` 重复生成物，以及生成路由表与若干框架源注释里的 world/scene/login 示例。本切片把这些**淘汰掉**，并纠正一处随之而来的陈旧分配注释。

## 2. 现状核实（已逐项 grep/读码/探针确认，2026-06-05 @ main 32b3518）

- `src/servers/dbsvr/`、`src/servers/loginsvr/`：**仅含 `.gitkeep`**，无任何 `.go`；全仓零引用。
- `protocal/world.proto`：定义 worldsvr/scenesvr 示例消息（`CS_ClaimReward_Req` 3001 / `SC_ClaimReward_Rsp` 3002 / `CS_SyncPos_OneWay` 3003 / `RPC_EnterMap_Req` 3101 / `RPC_EnterMap_Rsp` 3102）。
- `protocal/gen/world/world.pb.go`：**零 Go importer**（全仓唯一匹配是文件自身的 `go_package` 字符串）= 纯死代码。
- `protocal/gen/protocal/cluster.pb.go`：**零 importer 的孤儿重复生成物**；规范 cluster pb 是 `src/framework/cluster/pb/cluster.pb.go`（被 10+ 文件 import）。从未链入任何 binary，删除安全（无重复 proto 注册风险）。
- `protocal/gen/routes/routes.go`（生成物）：`MsgRouteTable` 含 3001/3003/3101、`ForwardTable` 含 3001→worldsvr / 3003→worldsvr / 3101→scenesvr。
- `tools/gen_routes` 是**纯文本/正则解析** `protocal/*.proto`（跳过 options.proto/cluster.proto；不 import 任何 gen 包、不需要 protoc）；删除 `world.proto` 后 `go run ./tools/gen_routes` 自动重生成不含 world/scene 条目的 routes.go。
- **无任何测试/运行代码依赖** world/scene 示例路由（grep `claimreward|syncpos|entermap|WorldHandler|SceneHandler|worldsvr(作路由)|3001|3003|3101` 在 `_test.go` 与业务代码命中为 0；唯一命中是注释、生成物 descriptor、以及 `logger`/`timewheel` 里**无关**的 `scenesvr` 示例字符串）。
- `protocal/options.proto:13-17` msg_id 区段分配注释**陈旧**：`2000-2999: loginsvr`，但 lobbysvr 实占 2001-2042（loginsvr 已并入 lobby）；`3000-3999: worldsvr` / `4000-4999: scenesvr` 是即将释放的区段。
- `protocal/cluster.proto:10`：`server_type_name` 字段注释示例为 `如 "worldsvr"`。
- **toolchain 探针（决定 D-A1 修订）**：committed `options.pb.go`/`cluster.pb.go` 由 protoc-gen-go **v1.36.11** 生成；沙箱仅有 v1.33.0（`github.com/golang/protobuf v1.5.4`）+ protoc 29.1。无源改动空跑重生 `options.pb.go` 即产生 27 增/12 删 churn（生成器降级 v1.36.11→v1.33.0、去 `unsafe` import、`rawDesc` 由拼接字符串重排为 `[]byte` 数组）。⇒ 沙箱**无法**对 `.pb.go` 做「仅注释」的清洁重生。

## 3. 范围内

1. **删占位目录**：`src/servers/dbsvr/`、`src/servers/loginsvr/`。
2. **删示例协议**：`protocal/world.proto`、`protocal/gen/world/`（整目录）。
3. **删孤儿重复生成物**：`protocal/gen/protocal/cluster.pb.go`（零 importer，连带空 `protocal/gen/protocal/` 目录消失）。
4. **重生成路由表**：`go run ./tools/gen_routes` → `protocal/gen/routes/routes.go` 自动移除 3001/3003/3101 的 World/Scene 条目（`MsgRouteTable` + `ForwardTable`）。**纯文本解析，无需 protoc。**
5. **纠正 msg_id 区段注释（仅 `.proto` 源）**（`protocal/options.proto`）：
   - `2000-2999: loginsvr` → `2000-2999: lobbysvr`（纠陈旧 bug，反映 loginsvr 并入 lobby 的事实）。
   - `3000-3999: worldsvr` / `4000-4999: scenesvr` → 标为「（已释放，预留扩展）」。
6. **框架示例注释改到目标拓扑**（决策 D-A2，纯注释、强相关项）：
   - 手写源（最终）：`src/framework/application/options.go`、`src/framework/application/application.go`、`src/framework/application/builder.go`、`src/framework/session/session.go` 中的 `worldsvr`/`scenesvr`/`loginsvr` 示例，改为 `gatesvr`/`lobbysvr`/`routersvr`/`onlinesvr`/`matchsvr`/`roomsvr` 目标拓扑示例。
   - `.proto` 源（仅源）：`protocal/cluster.proto:10` 的 `如 "worldsvr"` → 目标拓扑示例（如 `"lobbysvr"`）。
7. 清掉 1–6 引入的孤儿（未用 import/常量），若有。

## 4. 范围外 / 明确排除

- **任何生成 `.pb.go` 的注释镜像**（`protocal/gen/options/options.pb.go`、`src/framework/cluster/pb/cluster.pb.go`）：因沙箱缺 v1.36.11 toolchain，清洁重生不可达（§2 探针），**延 §C 文档同步切片**统一处理（连同 architecture/network/cluster/development 的 world/scene 改写；下次正规 v1.36.11 重生时由 `.proto` 源注释自愈）。本切片**不触碰任何生成 `.pb.go` 内容**，除 `routes.go`（文本重生）与删除上述孤儿。
- **`discovery.go` 的 `world-{worldID}` etcd key 命名空间绝不动**：`world-%d` 是 worldID（游戏"区"分区）概念，与已淘汰的 worldsvr 服务无关，是活代码。
- **`logger`（rotate.go / example_*_test.go）与 `timewheel` 里的 `scenesvr`**：无关的任意示例字符串（日志 basename / 帧驱动主循环示例），非旧服务引用，不碰。
- 根目录文档体系（architecture/network/cluster/development）的 world/scene 改写：留 §C 文档同步切片。
- §9.4 框架风险 backlog（B 系列）、派生残余 ②③④⑤。
- 既有其他死代码不主动删（CLAUDE.md「既有不主动删，提出来即可」）。

## 5. 关键决策（已与用户敲定）

- **D-A1（修订）**：原「借 protoc 重生 options.pb.go/cluster.pb.go 保源↔生成一致」经探针证实在沙箱会致生成器降级 churn（§2）→ 改为**仅改 `.proto`/`.go` 源注释**，生成 `.pb.go` 注释镜像**延 §C**（下次正规 v1.36.11 重生自愈）。本切片不动任何生成 `.pb.go` 内容（除 routes.go 文本重生）。
- **D-A2**：强相关框架源示例注释（application/session/builder/options/cluster.proto 源）**顺手改到目标拓扑**，避免 §C 时重翻；排除 logger/timewheel 无关示例。
- **D-A3**：**轻量化流程**——spec（本文件，短）+ impl-plan（短，~3–5 Task）合一个 docs PR → code 一个 PR；Task 轻评审（以删除/重生成为主，靠全量 build/vet/test 绿 + routes diff 断言保正确），不走 P4/P5-① 的核心 Task 双评审 + 整支终审。
- **D-A4（新增）**：孤儿 `protocal/gen/protocal/cluster.pb.go`（零 importer 重复生成物）纳入本切片删。

## 6. 验收标准

1. `go build ./...`、`go vet ./...`、`go vet -tags integration ./...`、`go test ./... -race` **全绿**，无悬挂引用。
2. `protocal/gen/routes/routes.go` **不再含** 3001/3003/3101 任一条目、不含 `WorldHandler`/`SceneHandler`/`worldsvr`/`scenesvr`。
3. `go run ./tools/gen_routes` **幂等**：清理后再跑一次，routes.go 无 diff。
4. 全仓 grep：无 `gen/world` 引用、无 `dbsvr`/`loginsvr` 目录或引用、无 `world.proto`、无 `protocal/gen/protocal/`。
5. `protocal/options.proto` 与 `protocal/cluster.proto` **源**注释反映目标拓扑（lobbysvr 占 2xxx、3xxx/4xxx 释放、字段示例非 worldsvr）。
6. **生成 `.pb.go` 除 `routes.go` 外零改动**（`git diff --stat` 不含 options.pb.go / src/framework/cluster/pb/cluster.pb.go 等任何 churn）；本切片不引入 toolchain 降级。

## 7. 风险

- **极低**。唯一实质风险「测试/代码依赖 world/scene 示例路由 fixture」——已核实为零（§2）。
- gen_routes 是文本解析，删 world.proto 后须**实跑** `go run ./tools/gen_routes` 重生成（不可手改 routes.go），并校验幂等。
- **已知非缺陷**：源 `.proto`/`.go` 注释与其生成 `.pb.go` 镜像在本切片后暂存 inert 注释 drift（如 options.pb.go 仍写 worldsvr）——延 §C 修订（自愈），不影响功能/wire/路由。

## 8. 实现步骤概览（细节见 impl-plan）

删占位目录 + 孤儿生成物（dbsvr/loginsvr/world.proto/gen/world/gen/protocal）→ 跑 gen_routes 重生成 routes.go（断言无 world/scene + 幂等）→ 改 options.proto / cluster.proto 源注释 → 改框架手写源示例注释 → 全量 build/vet/test -race + 「生成 .pb.go 零 churn」断言。
